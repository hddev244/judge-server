Judge Server API

Hệ thống chấm bài lập trình tự động hỗ trợ C++, Java, Python. Gửi code — nhận verdict trong vài giây. Hỗ trợ test thử trên test mẫu và nộp bài chính thức.

🌐 Base URL http://<server-ip>:8080
📋 Version v1
🔑 Auth X-API-Key (header)
🚀Quickstart — 3 bước

Lấy API Key

Liên hệ admin hoặc vào Admin Panel để tạo key. Key có dạng sk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx.

Test thử với test mẫu (không lưu)

bash
curl -X POST http://<server-ip>:8080/api/v1/submissions/test \
  -H "X-API-Key: YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"problemId":1,"language":"cpp","sourceCode":"..."}'

Trả kết quả ngay lập tức — chỉ chạy test case mẫu, không lưu vào DB.

Nộp bài chính thức

bash
# 1. Nộp
curl -X POST http://<server-ip>:8080/api/v1/submissions \
  -H "X-API-Key: YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"problemId":1,"language":"cpp","sourceCode":"..."}'
# → {"submissionId":"sub_xxxxxxxxxx","status":"PENDING",...}

# 2. Lấy kết quả (poll đến khi status != PENDING/JUDGING)
curl -H "X-API-Key: YOUR_KEY" \
  http://<server-ip>:8080/api/v1/submissions/sub_xxxxxxxxxx
🔑Xác thực

Mọi request cần header X-API-Key. Không có key hợp lệ → 401 Unauthorized.

Header — thêm vào mọi request
X-API-Key: sk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
⚠️ API Key là thông tin bí mật. Không commit vào source code hay chia sẻ công khai.
JavaScript / Node.js client
JavaScript
const JUDGE_BASE = "http://<server-ip>:8080";
const API_KEY    = "your_api_key";
const headers    = { "X-API-Key": API_KEY, "Content-Type": "application/json" };

// Test thử (đồng bộ, không lưu DB)
async function testRun(problemId, language, sourceCode) {
  const res = await fetch(`${JUDGE_BASE}/api/v1/submissions/test`, {
    method: "POST", headers,
    body: JSON.stringify({ problemId, language, sourceCode })
  });
  return res.json(); // kết quả ngay lập tức
}

// Nộp bài (async, cần poll)
async function submitAndWait(problemId, language, sourceCode) {
  const { submissionId } = await (await fetch(`${JUDGE_BASE}/api/v1/submissions`, {
    method: "POST", headers,
    body: JSON.stringify({ problemId, language, sourceCode })
  })).json();

  while (true) {
    const r = await (await fetch(`${JUDGE_BASE}/api/v1/submissions/${submissionId}`,
      { headers })).json();
    if (!["PENDING", "JUDGING"].includes(r.status)) return r;
    await new Promise(r => setTimeout(r, 2000));
  }
}
Python
import requests, time

JUDGE_BASE = "http://<server-ip>:8080"
HEADERS    = {"X-API-Key": "your_api_key"}

def test_run(problem_id, language, source_code):
    """Test thử — trả kết quả ngay, không lưu DB"""
    return requests.post(
        f"{JUDGE_BASE}/api/v1/submissions/test",
        headers={**HEADERS, "Content-Type": "application/json"},
        json={"problemId": problem_id, "language": language, "sourceCode": source_code}
    ).json()

def submit_and_wait(problem_id, language, source_code):
    """Nộp bài — lưu DB, chờ kết quả qua polling"""
    sub_id = requests.post(
        f"{JUDGE_BASE}/api/v1/submissions",
        headers={**HEADERS, "Content-Type": "application/json"},
        json={"problemId": problem_id, "language": language, "sourceCode": source_code}
    ).json()["submissionId"]

    while True:
        r = requests.get(f"{JUDGE_BASE}/api/v1/submissions/{sub_id}", headers=HEADERS).json()
        if r["status"] not in ("PENDING", "JUDGING"):
            return r
        time.sleep(2)

⚖️Custom Checker (Special Judge)

Mặc định server so sánh output chính xác từng ký tự (EXACT). Với bài chấp nhận nhiều đáp án (floating point, sắp xếp tùy ý...) bạn cần custom checker.

💡 Checker được compile 1 lần và lưu lại. Mỗi lần chấm, checker chạy trong Docker với 3 đối số: <input> <expected> <actual>. Exit code 0 = AC, khác = WA.
POST /api/v1/admin/problems/{id}/checker Admin Upload & compile checker (multipart)
Form fieldTypeMô tả
languagestringcpp · java · python
sourcefileSource code của checker
checker.cpp — so sánh số thực (sai số 1e-6)
#include <bits/stdc++.h>
using namespace std;
int main(int argc, char* argv[]) {
    ifstream expected(argv[2]), actual(argv[3]);
    double a, b;
    expected >> a; actual >> b;
    return abs(a - b) < 1e-6 ? 0 : 1;
}
bash — Upload checker
curl -X POST http://<server-ip>:8080/api/v1/admin/problems/1/checker \
  -H "X-API-Key: ADMIN_KEY" \
  -F "language=cpp" \
  -F "source=@checker.cpp"
DELETE /api/v1/admin/problems/{id}/checker Admin Gỡ checker, chuyển về EXACT match

Trả về ProblemResponse với checkerType: "EXACT".

🔔Webhook

Truyền callbackUrl khi nộp bài. Sau khi chấm xong, server POST kết quả về URL đó. Test Run không gửi webhook.

Payload — POST đến callbackUrl của bạn
{
  "submissionId": "sub_39a8bdc114",
  "status":       "AC",
  "score":        100,
  "timeMs":       260,
  "testResults": [...]
}
Xác minh chữ ký HMAC-SHA256

Header X-Judge-Signature — HMAC-SHA256 của payload, key là chính callbackUrl của bạn.

Node.js
const crypto = require('crypto');
function verify(payload, signature, callbackUrl) {
  const expected = crypto.createHmac('sha256', callbackUrl)
    .update(payload).digest('hex');
  return crypto.timingSafeEqual(Buffer.from(expected), Buffer.from(signature));
}
💡 Webhook thất bại → retry tự động tối đa 3 lần, backoff: 30s → 60s → 120s.

Sandbox constraints (mỗi lần chạy)
  • Network: Bị chặn hoàn toàn (--network none)
  • CPU: 0.5 core, hard limit theo timeLimitMs của problem
  • Memory: Giới hạn theo memoryLimitKb của problem
  • PID: Tối đa 64 processes
  • Filesystem: Read-only, chỉ /tmp (64 MB) là writable
