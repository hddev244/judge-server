Thêm ngôn ngữ lập trình mới

Judge Server không hardcode ngôn ngữ. Mỗi ngôn ngữ chỉ là 4 dòng config trong application.yml — không cần sửa code Java, không cần build lại.

⚙️Cơ chế hoạt động

Hiểu flow này sẽ giúp bạn điền đúng config ngay lần đầu.

Server nhận code và tạo thư mục làm việc

Source code được lưu vào file có tên source-file bên trong thư mục /tmp/judge/<job-id>/ trên máy chủ.

Compile (nếu có)

Server chạy compile-cmd bên trong container Docker (image = image), mount thư mục làm việc vào /code. Nếu exit code ≠ 0 → CE (Compile Error).

Lệnh Docker thực sự chạy khi compile
docker run --rm --network none \
  -v /tmp/judge/abc123:/code \
  -w /code \
  <image> /bin/sh -c "<compile-cmd>"

Chạy từng test case

Với mỗi test case, server chạy run-cmd trong container (thư mục code mount read-only). Input được feed qua stdin. Output của chương trình đọc từ stdout.

Lệnh Docker thực sự chạy khi run
docker run --rm --network none \
  --memory 256m --memory-swap 256m \
  --cpus 0.5 --pids-limit 64 \
  --read-only --tmpfs /tmp:size=64m \
  -v /tmp/judge/abc123:/code:ro \
  -v /data/problems/1/cases/1.in:/input.txt:ro \
  -w /code \
  <image> /bin/sh -c "timeout 2 <run-cmd> < /input.txt"
ℹ️ Working directory là /code. Tất cả lệnh compile và run đều chạy từ đây. Source file nằm tại /code/<source-file>, binary sau compile cũng nằm ở /code.
📋Giải thích từng field

Mỗi ngôn ngữ có 4 field trong application.yml:

image

Docker image chứa compiler/runtime. Server pull image này từ Docker Hub để chạy code.

Dùng tag cụ thể (vd: gcc:13) thay vì latest để tránh thay đổi ngoài ý muốn.

🔗 Tìm image tại hub.docker.com — search tên ngôn ngữ, chọn official image, ưu tiên tag slim hoặc alpine để nhẹ hơn.
source-file

Tên file mà server dùng để lưu source code.

Phải khớp với những gì compiler/runtime mong đợi. Ví dụ:

  • solution.go — Go compiler đọc file .go
  • Solution.java — Java yêu cầu tên file = tên class (Solution)
  • solution.py — Python không có yêu cầu đặc biệt, đặt tên gì cũng được
compile-cmd

Lệnh shell để compile source code. Chạy bên trong container tại /code.

Để trống ("") nếu ngôn ngữ không cần compile (Python, Ruby, PHP, Node.js...).

⚠️ Lệnh phải tạo ra binary/bytecode cũng tại /code. Ví dụ Go: go build -o solution solution.go → tạo /code/solution
run-cmd

Lệnh shell để chạy chương trình. Stdin là input, stdout là output của bài.

Hỗ trợ placeholder {mem} — được thay bằng memory limit (MB) của problem trước khi chạy. Dùng cho JVM để set heap:

Ví dụ dùng {mem} cho JVM
run-cmd: "java -Xmx{mem}m -cp /code Solution"
# Nếu problem có memoryLimitKb=262144 (256MB),
# lệnh thực tế sẽ là: java -Xmx256m -cp /code Solution
🚀Các bước thêm ngôn ngữ

Pull Docker image về máy chủ

Image cần có sẵn trên host mới chạy được. Chạy lệnh này trên máy chủ (hoặc thêm vào scripts/init-images.sh):

bash
sudo docker pull golang:1.22-alpine

Thêm config vào application.yml

src/main/resources/application.yml
judge:
  languages:
    cpp:      # ... giữ nguyên
    java:     # ... giữ nguyên
    python:   # ... giữ nguyên
    go:
      image: golang:1.22-alpine
      source-file: solution.go
      compile-cmd: "go build -o solution solution.go"
      run-cmd: "./solution"

(Tùy chọn) Thêm biến môi trường vào .env

Nếu muốn đổi image linh hoạt mà không sửa file YAML:

.env
JUDGE_LANG_GO_IMAGE=golang:1.22-alpine
application.yml — dùng biến môi trường
    go:
      image: ${JUDGE_LANG_GO_IMAGE:golang:1.22-alpine}
      source-file: solution.go
      compile-cmd: "go build -o solution solution.go"
      run-cmd: "./solution"

Rebuild và restart server

bash
docker compose build --no-cache judge-api
docker compose up -d judge-api

Kiểm tra bằng cách nộp bài thử

bash — test run Hello World
curl -X POST http://localhost:8080/api/v1/submissions/test \
  -H "X-API-Key: YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"problemId":1,"language":"go","sourceCode":"package main\nimport \"fmt\"\nfunc main(){fmt.Scan(&a,&b);fmt.Println(a+b)}"}'
⚠️ Nếu image chưa được pull → server sẽ tốn thêm thời gian pull lần đầu. Nên chạy docker pull trước khi khởi động.
